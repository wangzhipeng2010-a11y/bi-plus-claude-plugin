# 后续 Skill 扩展建议

这些只是方向，不在 MVP 实现。

## check-permission

用户问“我能不能发送这个报表”“我有没有这个数据权限”时，调用后台只读接口查看当前状态。

边界：只展示后台返回的权限状态，不自行判断权限。

## request-more-auth

用户没有发送权限或目录权限时，引导用户发起加权申请。

边界：只整理申请信息并调用后台入口，不实现审批流。

## send-feishu-message

把 Claude Code 里生成的简短结论发送到用户飞书。

边界：仍由后台服务发送，不直接调用飞书 API。

## list-recent-results

列出后台认可的最近结果文件，帮助用户选择要发送的文件。

边界：不扫描用户全盘，不推断敏感目录，只展示后台返回的允许结果。

## explain-delivery-status

查询一次发送任务的状态，例如已发送、失败、等待审计。

边界：只解释状态，不修改后台任务。
